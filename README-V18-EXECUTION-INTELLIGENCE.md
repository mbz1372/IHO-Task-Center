# IHO Task Center V18 — راهنمای اجرایی

## هسته تصمیم‌گیری

نسخه ۱۸ حول سه موجودیت اصلی طراحی شده است:

- **هتل:** پرونده، مسئولان، قرارداد، ظرفیت، Provider، ارتباطات و رویدادها.
- **کارشناس:** مسئولیت هتل‌ها، حجم کار، تسک‌ها، فعالیت‌ها، اهداف و KPI.
- **اقدام:** دلیل، اولویت، مسئول، موعد، منبع ایجاد و نتیجه قابل‌ردیابی.

## ترتیب راه‌اندازی

1. فایل `database/SUPABASE-V18-EXECUTION-INTELLIGENCE.sql` را اجرا کنید.
2. در بخش «نقش‌ها و دسترسی‌ها»، دسترسی‌های جدید را برای نقش‌های سفارشی فعال کنید.
3. از «مالکیت پرونده هتل» مسئولان هر هتل را تعیین کنید.
4. «مرکز فرمان اقدام» را بررسی و برای هشدارهای مهم تسک بسازید.
5. در «اتوماسیون» قوانین پیش‌فرض را به مسئول مناسب متصل و ابتدا اسکن آزمایشی اجرا کنید.
6. پس از یک دوره آزمایشی، KPIها را با اهداف رسمی تیم مقایسه کنید.

## قواعد عملیاتی پیشنهادی

- هر هتل فعال حداقل یک مالک پرونده داشته باشد.
- تسک بدون مسئول و موعد ایجاد نشود.
- ارتباط دارای پیگیری بعدی، هم‌زمان تسک پیگیری بسازد.
- اجرای اتوماسیون ابتدا با سقف ۲۵ رکورد انجام شود.
- KPI در شروع فقط برای مشاهده و کالیبراسیون استفاده شود؛ سپس مبنای ارزیابی قرار گیرد.

