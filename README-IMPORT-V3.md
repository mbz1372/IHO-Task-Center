# Import V3

## فایل هتل‌ها
فایل واقعی `All hotels.xlsx` مستقیماً پشتیبانی می‌شود. اتصال رکورد با `کد هتل` و ذخیره شناسه به صورت `hotel-{code}` انجام می‌شود.

## فایل تخصیص کارشناس
فایل واقعی `assignment hotel.xlsx` ستون‌های زیر را دارد:
- `نام هتل`
- `taskId`
- `First name`

چون کد هتل ندارد، تطبیق با نام هتل نرمال‌شده انجام می‌شود. taskId=1 کارشناس ظرفیت و taskId=2 کارشناس نرخ است. نتیجه Import تعداد تطبیق موفق، نام پیدا نشده، نام تکراری و ردیف ناقص را نمایش می‌دهد.

## فرایند داخل پلتفرم
در تب «عملیات و پیگیری» برای هر هتل ثبت می‌شود:
- وضعیت آنلاین‌سازی
- مسئول آنلاین‌سازی
- شخصی که آنلاین کرده
- تاریخ آنلاین‌شدن
- مسئول و تاریخ پیگیری
- مسئول، وضعیت و نوبت بعدی Care
- یادداشت عملیاتی

قبل از Deploy فایل `SUPABASE-MIGRATION-V3.sql` را یک‌بار در SQL Editor اجرا کنید.
