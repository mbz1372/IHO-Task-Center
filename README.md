# IranHotel OS V18 — Execution Intelligence

نسخه ۱۸ هسته IHO Task Center را به یک سیستم هتل‌محور، کارشناس‌محور و اقدام‌محور تبدیل می‌کند و با همان Supabase و متغیرهای محیطی نسخه قبلی کار می‌کند.

## قابلیت‌های جدید نسخه ۱۸

- مرکز فرمان اقدام برای پاسخ سریع به سؤال «کدام هتل، چرا، توسط چه کسی و تا چه زمانی نیازمند اقدام است؟»
- امتیاز سلامت و تشخیص خودکار هتل‌های بدون ظرفیت، قرارداد ناقص، Provider نامناسب، مسئول نامشخص و تسک معوق.
- مالکیت چندنقشی پرونده هتل شامل مدیر شهر، اکانت منیجر، کارشناس نرخ و کارشناس ظرفیت با حفظ تاریخچه تخصیص.
- مرکز KPI واقعی کارشناسان شامل نرخ تکمیل، انجام به‌موقع، عقب‌افتادگی، فعالیت و امتیاز ترکیبی.
- Timeline واقعی ارتباطات هتل با ثبت تماس، پیام، جلسه، نتیجه و ساخت خودکار تسک پیگیری.
- موتور اجرای اتوماسیون با اسکن شرایط، جلوگیری از تسک تکراری، سقف اجرای هر قانون و تاریخچه اجرا.
- Event Stream هتل برای ثبت خودکار ساخت تسک، تغییر وضعیت و ارتباطات در تاریخچه هتل.
- مدل داده آماده اتصال نرخ، ظرفیت، فروش، رزرو و KPIهای دوره‌ای.
- دسترسی‌های مستقل برای مرکز فرمان، مالکیت هتل و مرکز KPI.

## ارتقا از نسخه ۱۷

پس از Migrationهای قبلی، فایل زیر را یک‌بار در Supabase SQL Editor اجرا کنید:

```text
database/SUPABASE-V18-EXECUTION-INTELLIGENCE.sql
```

Migration نسخه ۱۸ idempotent است و اجرای مجدد آن داده‌های موجود را حذف نمی‌کند.

## اصلاحات کلیدی

- نوار کناری واقعاً باز و بسته می‌شود؛ حالت انتخاب‌شده در مرورگر حفظ می‌شود.
- حالت شب در کل رابط، فرم‌ها، جدول‌ها، نمودارها و پنل‌ها اعمال می‌شود.
- فونت Vazirmatn Variable و مقیاس خواناتر تایپوگرافی در کل سیستم.
- داشبورد مدیریتی جدید با نمودار روند تسک، وضعیت تسک‌ها، Providerها و عملکرد تیم.
- آمار سوپر اپ هتل بر اساس دیتابیس اصلی و فقط هتل‌های «در حال همکاری» محاسبه می‌شود.
- «کل دیتابیس» و «هتل‌های قابل فروش» به‌صورت دو شاخص مستقل نمایش داده می‌شوند.
- فیلتر پیش‌فرض پرونده هتل‌ها روی هتل‌های در حال همکاری است و امکان مشاهده کل دیتابیس وجود دارد.
- کش آمار نسخه‌های قبلی کنار گذاشته شده تا داده قدیمی نمایش داده نشود.
- دکمه برجسته «مدیریت و پاک‌سازی داده» برای سوپر ادمین در خود سوپر اپ هتل.
- نماهای کاری ذخیره‌شده، عملیات گروهی، اسناد و قراردادها بازطراحی شده‌اند.
- تاریخ‌های شمسی Excel بدون تبدیل اشتباه نمایش داده می‌شوند.
- قراردادها فقط برای هتل‌های در حال همکاری تحلیل می‌شوند و «فاقد تاریخ قرارداد» جداست.
- Import اکسل همه ردیف‌های دارای کد هتل را نگه می‌دارد؛ ردیف بدون نام با عنوان موقت مشخص وارد می‌شود.
- XLSX، نمودارها و سوپر اپ به‌صورت Lazy Load بارگذاری می‌شوند.

## منطق آمار هتل

- کل هتل‌ها: تعداد واقعی جدول `ihos_hotels`.
- قابل فروش: فقط `وضعیت همکاری` شامل «در حال همکاری».
- ۱۰۰٪ آنلاین: نرخ و ظرفیت هر دو از API Provider یا توسط خود هتل تأمین شوند.
- ظرفیت آنلاین: ظرفیت آنلاین است اما نرخ هنوز نیازمند کارشناس است.
- نرخ آنلاین: نرخ آنلاین است اما ظرفیت هنوز نیازمند کارشناس است.
- کارشناس‌محور: نرخ یا ظرفیت به کارشناس تخصیص دارد و هیچ‌کدام آنلاین کامل نیست.
- آفلاین: منبع آنلاین یا کارشناس مشخصی ثبت نشده است.
