# اصلاح محدودیت ۱۰۰۰ ردیف Supabase

## مشکل
PostgREST/Supabase به طور معمول در هر درخواست Select حداکثر ۱۰۰۰ ردیف برمی‌گرداند. در نسخه قبلی، جدول `ihos_hotels` با یک Select ساده خوانده می‌شد؛ بنابراین حتی اگر ۹۱۵۴ ردیف ذخیره شده بود، فقط ۱۰۰۰ ردیف در برنامه دیده می‌شد.

## اصلاحات
- دریافت تمام هتل‌ها به صورت صفحه‌ای با `range()` و اندازه صفحه ۱۰۰۰ ردیف
- ذخیره Excel در دسته‌های ۵۰۰ ردیفی
- نمایش خطای دقیق هر Batch
- Sync مجدد جدول هتل‌ها بعد از پایان Import

## بررسی تعداد واقعی در Supabase
در SQL Editor اجرا کنید:

```sql
select count(*) from public.ihos_hotels;
```
