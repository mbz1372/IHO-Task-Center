# IranHotel OS V17 — Enterprise Operations Intelligence

## امکانات جدید

- ماتریس پوشش هتل‌های Providerها و ورود Excel برای هر Provider
- پیشنهاد بهترین Provider برای مهاجرت هر هتل بر اساس پوشش و API نرخ/ظرفیت
- مرکز تأیید و کنترل کیفیت برای تایید، بازگشت برای اصلاح و بستن تسک‌ها
- منوی گروهی با زیرمنوهای باز و بسته‌شونده و حفظ وضعیت کاربر
- پیام‌رسان داخلی Supabase با پیام تیمی، مستقیم، سازمانی و اتصال پیام به هتل
- تقویم ماهانه، هفتگی و فهرستی با تسک، رویداد و یادآور
- قوانین SLA قابل ویرایش برای نوع و دسته تسک
- پیشنهاد خودکار Deadline تسک از SLA فعال
- چرخه و سلامت فرایندی هتل بدون وابستگی به ظرفیت فیزیکی
- پروفایل مالی قرارداد، مدل تسویه و پیش‌بینی دسته ریسک هتل
- نقشه شماتیک پوشش مناطق و بار کاری تیم‌ها

## راه‌اندازی

1. فایل `database/SUPABASE-V17-ENTERPRISE-MODULES.sql` را در Supabase SQL Editor اجرا کنید.
2. Environment Variableهای نسخه قبلی را بدون تغییر نگه دارید.
3. پروژه را Deploy کنید.
4. برای ورود پوشش Provider از مسیر `سوپر اپ هتل ← پوشش Providerها` استفاده کنید.

## مرکز تأیید چیست؟

این مرکز Quality Gate عملیات است. کارشناس پس از پایان کار، وضعیت تسک را روی «ارسال برای تایید» می‌گذارد. مدیر آن را بررسی می‌کند و یکی از دو تصمیم را می‌گیرد:

- تایید شده
- نیازمند اصلاح

به این ترتیب تسک بدون کنترل کیفیت مستقیماً بسته نمی‌شود.
